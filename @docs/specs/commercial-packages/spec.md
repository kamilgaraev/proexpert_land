# Коммерческий конструктор МОСТ

## Цель

Заменить тарифную матрицу единой коммерческой моделью: бесплатная ограниченная база, десять подключаемых бизнес-пакетов, полный комплект и корпоративный уровень.

## Источник истины

Состав, цены, статусы доступа, расчёт доплаты, периоды и рекомендации определяет backend МОСТ. Маркетинговые данные используются только для публичного SSR/SEO и обязаны совпадать с серверным каталогом. Личный кабинет всегда получает данные через защищённый API.

## Пользовательский поток

1. Регистрация создаёт бесплатную базу без платного доступа.
2. Пользователь с правом просмотра видит доступный контур и коммерческие состояния.
3. Пользователь с правом управления может запустить один 72-часовой trial для каждого пакета.
4. Пользователь собирает набор пакетов и получает серверный расчёт. При восьми и более пакетах система рекомендует полный комплект, но не выбирает его автоматически.
5. Положительная доплата проводится прямым платежом через ЮKassa. Внутренний баланс не является способом оплаты пакетов.
6. Доступ включается только после подтверждённого сервером успешного платежа.
7. Сохранённый способ оплаты используется для автопродления при явном согласии пользователя.
8. При временной ошибке оплаты начинается семидневный льготный период с ежедневной ночной попыткой. Расчётная дата не переносится: поздняя оплата продлевает доступ только до прежней следующей даты.
9. Отключение пакетов без доплаты применяется со следующего расчётного периода и не отбирает уже оплаченный доступ.

## Видимость и уведомления

Коммерческие уведомления видят только пользователи текущей организации с `billing.view` или `billing.manage` в центре уведомлений личного кабинета. Ссылки ведут на `/dashboard/billing`. Админские экраны, списки платежей и админские уведомления не создаются.

## Корпоративный уровень

Корпоративное предложение строится поверх полного комплекта и может включать несколько организаций, сводную отчётность, SSO, аудит, API/webhooks, интеграции 1С/ERP/BI, SLA, выделенного менеджера, миграцию и обучение. Индивидуальность реализуется настройками, правами, feature flags и интеграциями без отдельной ветки SaaS.

## Ограничения

- Нет совместимости со старыми тарифами и сохранения тестовых подписок.
- Нет клиентской активации доступа по redirect-параметрам.
- Нет прямого платежа пакетами из внутреннего баланса.
- Нет автоматического переключения на полный комплект.
- Изменение состава пакетов запрещено во время льготного периода.
- Секреты ЮKassa не хранятся в репозитории и не попадают в интерфейс или логи.

## Критерии приёмки

- Публичный сайт нигде не продаёт Start, Business, Profi или Enterprise Constructor.
- Личный кабинет покрывает trial, расчёт, оплату, ожидание webhook, успех, отмену, возврат, grace, окончание доступа и историю.
- Все мутации проверяют права и организацию, повторные запросы безопасны.
- Русские тексты сохранены в UTF-8, интерфейс адаптивен и доступен с клавиатуры.
- Backend и frontend проверки проходят без запуска локальных миграций и без frontend production build.
