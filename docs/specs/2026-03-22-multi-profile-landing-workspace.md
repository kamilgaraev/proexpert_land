# Multi-profile Landing Workspace

## Что меняется
- `workspace_profile` становится soft-orchestrator для навигации и стартового маршрута.
- Оплаченные и активные модули не скрываются, а только влияют на приоритет меню.
- Sidebar показывает текущий основной workspace и тип рабочего контура.

## UX-правила
- При первом заходе на `/dashboard` пользователь переводится в приоритетный workspace.
- Приоритет меню определяется комбинацией:
  - `workspace_profile.primary_profile`
  - активных модулей организации
- Обзор остается доступным вручную, но больше не является единственным стартовым экраном.

## Ограничения
- Никаких скрытых hard-gate по типу деятельности.
- `workspace_profile` не выдает права и не обходит permissions.
