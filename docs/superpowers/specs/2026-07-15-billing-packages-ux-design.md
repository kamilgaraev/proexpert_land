# UX пакетов и оплаты МОСТ

## Цель

Убрать неоднозначность между уже оплаченными пакетами и новым выбором пользователя. Страница должна сразу отвечать на три вопроса: что уже подключено, что можно добавить и сколько будет стоить конкретное изменение.

## Структура страницы

Страница сохраняет общий заголовок и сведения об оплате, а каталог делится на два независимых блока.

### Подключённые пакеты

Блок расположен перед каталогом и содержит только действующие оплаченные пакеты. Карточка показывает:

- название и цену в месяц;
- состояние «Подключён» либо «Отключится <дата>»;
- дату следующей оплаты;
- действие «Подробнее»;
- действие «Отключить со следующего периода».

Оплаченный пакет не отображается выбранной галочкой. Нажатие на отключение добавляет пакет в проект изменения, но не прекращает доступ сразу. До подтверждения пользователь может отменить это действие. После подтверждения карточка остаётся в подключённых и показывает дату будущего отключения.

### Добавить возможности

Блок содержит только неподключённые пакеты. В карточке используются действия «Добавить» и «Подробнее». После добавления карточка получает состояние «Добавлено к заказу» и действие отмены выбора. Пробный доступ остаётся доступен только для подходящих неподключённых пакетов.

Полный комплект показывается отдельной заметной карточкой над каталогом и не смешивается с отдельными пакетами.

## Расчёт изменения

Боковой блок больше не считает уже оплаченные пакеты новым выбором. Он показывает:

- текущую стоимость подключённых пакетов;
- добавляемые пакеты и их стоимость;
- пакеты, которые отключатся со следующего периода;
- сумму к оплате сейчас;
- стоимость следующего периода;
- следующую дату оплаты.

Если изменений нет, основное действие заменяется нейтральным состоянием «Изменений нет». При добавлении доступно действие «Перейти к оплате», при одном сокращении — «Запланировать отключение».

## Подробности пакета

«Подробнее» открывает боковую панель на широком экране и полноэкранную панель на мобильном. В ней показываются:

- название, назначение и цена пакета;
- основные результаты для бизнеса;
- модули пакета;
- русское название и короткое пользовательское описание каждого модуля;
- доступные действия с учётом состояния пакета.

Названия и описания модулей приходят из серверного каталога. Клиент не хранит отдельную копию этих данных.

## API-контракт

Элемент `modules` в ответе каталога пакетов меняется со списка технических идентификаторов на список объектов:

```json
{
  "slug": "schedule_management",
  "name": "Графики и планы",
  "description": "Планирование сроков, этапов и работ по объектам."
}
```

Сервер формирует эти данные из существующих определений модулей. Технические поля, внутренние классификации и служебные статусы в интерфейс не передаются.

## Состояния и ошибки

- Загрузка показывает каркас блоков без скачков компоновки.
- Ошибка подробностей не закрывает страницу и предлагает повторить загрузку.
- Недоступный или удалённый модуль отображается с названием из каталога пакета и безопасным кратким описанием, не ломая весь пакет.
- Запланированное отключение всегда сопровождается точной датой сохранения доступа.
- После оплаты или изменения состава каталог, сведения об оплате и навигационные доступы обновляются одним повторным чтением серверного состояния.

## Дополнительные улучшения

- Дата сокращается до вида «14 августа, 06:21» без повторения года и лишних служебных формулировок.
- Для уже настроенного автоплатежа не показывается повторное согласие без реальной необходимости.
- Карточки подключённых пакетов визуально спокойнее каталога, а новое добавление выделяется оранжевым цветом действия.
- Все интерактивные состояния доступны с клавиатуры и имеют понятные подписи для экранных дикторов.

## Проверка

- Backend-тест подтверждает русские названия и описания модулей в каталоге каждого пакета.
- Frontend-тесты проверяют раздельные списки, отсутствие галочек у оплаченных пакетов, расчёт только изменений, подробности модулей и отложенное отключение.
- TypeScript и ESLint проходят без ошибок.
- После деплоя проверяются широкая и мобильная компоновки, добавление пакета, просмотр подробностей и планирование отключения.
