# Обновление счётчика Яндекс Метрики

## Цель

Подключить счётчик Яндекс Метрики `110599591` ко всем публичным страницам маркетингового сайта МОСТ на доменах `1мост.рф` и `www.1мост.рф`, включая переходы внутри React SPA. Не отправлять аналитику из личного кабинета и внутренних пользовательских маршрутов.

## Область действия

Счётчик инициализируется только при одновременном выполнении условий:

- пользователь разрешил аналитические cookie;
- текущий хост — `1мост.рф` или `www.1мост.рф`;
- текущий маршрут относится к публичному маркетинговому сайту.

Счётчик не инициализируется на `lk.1мост.рф`, других поддоменах, локальных хостах, страницах авторизации и внутренних маршрутах личного кабинета.

## Инициализация

Используется единый идентификатор `110599591`. Параметры инициализации соответствуют новому коду счётчика:

- `ssr: true`;
- `webvisor: true`;
- `clickmap: true`;
- `ecommerce: "dataLayer"`;
- `accurateTrackBounce: true`;
- `trackLinks: true`.

`trackHash` не используется, поскольку просмотры SPA-маршрутов отправляются приложением явно.

## Учёт просмотров

Первый просмотр отправляется при инициализации счётчика. При последующих изменениях React Router location приложение вызывает `ym(110599591, 'hit', url, options)` после обновления заголовка страницы.

В событие перехода передаются:

- полный актуальный URL;
- актуальный `document.title`;
- URL предыдущего SPA-маршрута как `referer`.

Повторная отрисовка того же URL не должна создавать дублирующий просмотр.

## Цели и пользовательские события

Вспомогательные функции отправки целей и событий используют тот же единый идентификатор `110599591` и те же проверки согласия, домена и публичного маршрута.

## Проверка

Автоматические тесты должны подтверждать:

- использование нового идентификатора;
- включение счётчика на `1мост.рф` и `www.1мост.рф`;
- отключение на `lk.1мост.рф`, других поддоменах и внутренних маршрутах;
- отправку одного просмотра при SPA-переходе;
- отсутствие дубля для неизменившегося URL;
- использование нового идентификатора при отправке целей и событий.

После изменений выполняются целевые тесты, `npx tsc --noEmit` и ESLint для затронутых файлов. Сборка проекта не запускается.
